Configuration files in Python

July 27th, 2014

Most interesting programs need some kind of configuration:

  • Content Management Systems like WordPress blogs, WikiMedia and Joomla need
    to store the information where the database server is (the hostname) and
    how to login (username and password)
  • Proprietary software might need to store if the software was registered already
    (the serial key)
  • Scientific software could store the path to BLAS libraries

For very simple tasks you might choose to write these configuration variables
directly into the source code. But this is a bad idea when you upload the code
to GitHub.

I will explain some alternatives I got to know for Python.

Python Configuration File

The simplest way to write configuration files is to simply write a separate file
that contains Python code. You might want to call it something like
databaseconfig.py. Then you could add the line *config.py to your
.gitignore file to avoid uploading it accidentally.

A configuration file could look like this:

 1#!/usr/bin/env python
 2import preprocessing
 3mysql = {'host': 'localhost',
 4         'user': 'root',
 5         'passwd': 'my secret password',
 6         'db': 'write-math'}
 7preprocessing_queue = [preprocessing.scale_and_center,
 8                       preprocessing.dot_reduction,
 9                       preprocessing.connect_lines]
10use_anonymous = True

Within the actual code, you can use it like this:

1#!/usr/bin/env python
2import databaseconfig as cfg
3connect(mysql['host'], mysql['user'], mysql['password'])

The way you include the configuration might feel very convenient at a first
glance, but imagine what happens when you get more configuration variables.
You definitely need to provide an example configuration file. And it is hard
to resist the temptation to include code within the configuration file.

JSON

JSON is short for JavaScript Object
Notation. It is widespread and thus has good support for many programming
languages.

The configuration might look like this:

 1{
 2    "mysql":{
 3        "host":"localhost",
 4        "user":"root",
 5        "passwd":"my secret password",
 6        "db":"write-math"
 7    },
 8    "other":{
 9        "preprocessing_queue":[
10            "preprocessing.scale_and_center",
11            "preprocessing.dot_reduction",
12            "preprocessing.connect_lines"
13            ],
14        "use_anonymous":true
15    }
16}

You can read it like this:

1import json
2
3with open('config.json') as json_data_file:
4    data = json.load(json_data_file)
5print(data)

which outputs

1{u'mysql': {u'db': u'write-math',
2            u'host': u'localhost',
3            u'passwd': u'my secret password',
4            u'user': u'root'},
5 u'other': {u'preprocessing_queue': [u'preprocessing.scale_and_center',
6                                     u'preprocessing.dot_reduction',
7                                     u'preprocessing.connect_lines'],
8            u'use_anonymous': True}}

Writing JSON files is also easy. Just build up the dictionary and use

1import json
2with open('config.json', 'w') as outfile:
3    json.dump(data, outfile)

YAML

YAML is a configuration file format.
Wikipedia says:

YAML (rhymes with camel) is a human-readable data serialization format that
takes concepts from programming languages such as C, Perl, and Python, and
ideas from XML and the data format of electronic mail (RFC 2822). YAML was
first proposed by Clark Evans in 2001, who designed it together with
Ingy d√∂t Net and Oren Ben-Kiki.
It is available for several programming languages.

The file itself might look like this:

 1mysql:
 2    host: localhost
 3    user: root
 4    passwd: my secret password
 5    db: write-math
 6other:
 7    preprocessing_queue:
 8        - preprocessing.scale_and_center
 9        - preprocessing.dot_reduction
10        - preprocessing.connect_lines
11    use_anonymous: yes

You can read it like this:

 1import yaml
 2
 3with open("config.yml", 'r') as ymlfile:
 4    cfg = yaml.load(ymlfile)
 5
 6for section in cfg:
 7    print(section)
 8print(cfg['mysql'])
 9print(cfg['other'])

It outputs:

 1other
 2mysql
 3{'passwd': 'my secret password',
 4 'host': 'localhost',
 5 'db': 'write-math',
 6 'user': 'root'}
 7{'preprocessing_queue': ['preprocessing.scale_and_center',
 8                         'preprocessing.dot_reduction',
 9                         'preprocessing.connect_lines'],
10 'use_anonymous': True}

There is a yaml.dump method, so you can write the configuration the same way.
Just build up a dictionary.

YAML is used by the Blender project.

INI

INI files look like this:

 1[mysql]
 2host=localhost
 3user=root
 4passwd=my secret password
 5db=write-math
 6
 7[other]
 8preprocessing_queue=["preprocessing.scale_and_center",
 9                     "preprocessing.dot_reduction",
10                     "preprocessing.connect_lines"]
11use_anonymous=yes

ConfigParser

Basic example

The file can be loaded and used like this:

 1#!/usr/bin/env python
 2
 3import ConfigParser
 4import io
 5
 6# Load the configuration file
 7with open("config.yml") as f:
 8    sample_config = f.read()
 9config = ConfigParser.RawConfigParser(allow_no_value=True)
10config.readfp(io.BytesIO(sample_config))
11
12# List all contents
13print("List all contents")
14for section in config.sections():
15    print("Section: %s" % section)
16    for options in config.options(section):
17        print("x %s:::%s:::%s" % (options,
18                                  config.get(section, options),
19                                  str(type(options))))
20
21# Print some contents
22print("\nPrint some contents")
23print(config.get('other', 'use_anonymous'))  # Just get the value
24print(config.getboolean('other', 'use_anonymous'))  # You know the datatype?

which outputs

 1List all contents
 2Section: mysql
 3x host:::localhost:::<type 'str'>
 4x user:::root:::<type 'str'>
 5x passwd:::my secret password:::<type 'str'>
 6x db:::write-math:::<type 'str'>
 7Section: other
 8x preprocessing_queue:::["preprocessing.scale_and_center",
 9"preprocessing.dot_reduction",
10"preprocessing.connect_lines"]:::<type 'str'>
11x use_anonymous:::yes:::<type 'str'>
12
13Print some contents
14yes
15True

As you can see, you can use a standard data format that is easy to read and
write. Methods like getboolean and getint allow you to get the datatype
instead of a simple string.

Writing configuration

 1import os
 2configfile_name = "config.yaml"
 3
 4# Check if there is already a configurtion file
 5if not os.path.isfile(configfile_name):
 6    # Create the configuration file as it doesn't exist yet
 7    cfgfile = open(configfile_name, 'w')
 8
 9    # Add content to the file
10    Config = ConfigParser.ConfigParser()
11    Config.add_section('mysql')
12    Config.set('mysql', 'host', 'localhost')
13    Config.set('mysql', 'user', 'root')
14    Config.set('mysql', 'passwd', 'my secret password')
15    Config.set('mysql', 'db', 'write-math')
16    Config.add_section('other')
17    Config.set('other',
18               'preprocessing_queue',
19               ['preprocessing.scale_and_center',
20                'preprocessing.dot_reduction',
21                'preprocessing.connect_lines'])
22    Config.set('other', 'use_anonymous', True)
23    Config.write(cfgfile)
24    cfgfile.close()

results in

 2host = localhost
 3user = root
 4passwd = my secret password
 5db = write-math
 6
 7[other]
 8preprocessing_queue = ['preprocessing.scale_and_center', 'preprocessing.dot_reduction', 'preprocessing.connect_lines']
 9use_anonymous = True
10

XML

Seems not to be used at all for configuration files by the Python community.
However, parsing / writing XML is easy and there are plenty of possibilities to
do so with Python. One is BeautifulSoup:

 1from BeautifulSoup import BeautifulSoup
 2
 3with open("config.xml") as f:
 4    content = f.read()
 5
 6y = BeautifulSoup(content)
 7print(y.mysql.host.contents[0])
 8for tag in y.other.preprocessing_queue:
 9    print(tag)

where the config.xml might look like this:

 1<config>
 2    <mysql>
 3        <host>localhost</host>
 4        <user>root</user>
 5        <passwd>my secret password</passwd>
 6        <db>write-math</db>
 7    </mysql>
 8    <other>
 9        <preprocessing_queue>
10            <li>preprocessing.scale_and_center</li>
11            <li>preprocessing.dot_reduction</li>
12            <li>preprocessing.connect_lines</li>
13        </preprocessing_queue>
14        <use_anonymous value="true" />
15    </other>
16</config>

File Endings

File Endings give the user and the system an indicator about the content of a
file. Reasonable file endings for configuration files are

  • *config.py for Python files
  • *.yaml or *.yml if the configuration is done in YAML format
  • *.json for configuration files written in JSON format
  • *.cfg or *.conf to indicate that it is a configuration file
  • *.ini for ‚Äúinitialization‚Äù are quite widespread (see Wiki)
  • ~/.[my_app_name]rc is a VERY common naming scheme for configuration files on
    Linux systems. RC is a reference to an old computer system and means
    ‚Äúrun common‚Äù.

That said, I think I prefer *.conf. I think it is a choice that users
understand.

But you might also consider that *.ini might get opened by standard in a
text editor. For the other options, users might get asked which program they
want to use.
