Why? How can it be fixed to get the expected output? {% highlight c %}#define __STDC_FORMAT_MACROS #include #include int main() { uint64_t testvar; testvar = 1<<30; printf("2^30 = %" PRIu64 "\n", testvar); testvar = 1<<31; printf("2^31 = %" PRIu64 "\n", testvar); return 0; } {% endhighlight %} . . . . . . . . . . . . . . . . .

Answer

{% highlight text %}2^30 = 1073741824 2^31 = 18446744071562067968{% endhighlight %}

Expected output

{% highlight text %}2^30 = 1073741824 2^31 = 2147483648{% endhighlight %}

Explanation

1<<30 does a bitshift for int, not for uint64_t

How to fix it

Add a typecast: {% highlight c %}#define __STDC_FORMAT_MACROS #include #include int main() { uint64_t testvar; testvar = 1<<30; printf("2^30 = %" PRIu64 "\n", testvar); testvar = ((uint64_t) 1)<<31; printf("2^31 = %" PRIu64 "\n", testvar); return 0; } {% endhighlight %}
